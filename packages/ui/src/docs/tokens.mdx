# Tokens — the ions

Four tiers in `packages/ui/src/styles/globals.css`, all color in **oklch**:

```
composite / component ──▶ semantic ──▶ primitive
```

- **Primitive** (`--primitive-<hue>-<step>`) — context-free raw values. Defined in
  `:root` only, **never in `@theme`**: that's the enforcement trick, not a style
  preference — Tailwind generates *no utilities* for them, so a component
  structurally cannot consume a raw value by class. Never themed.
- **Semantic** (`--background`, `--muted`, `--status-*`, `--sla-breach`…) — roles
  that reference primitives. Light/dark remaps happen at **this tier only**: `.dark`
  re-points roles; pure aliases (`--status-breached`, `--destructive`, `--ring`)
  are declared once and chase their targets through the remap.
- **Component** (`--<component>-<part>`) — the earning rule: minted only when two
  consumers need *different values for the same semantic role*. **Zero exist.** A
  knob nobody overrides is dead weight.
- **Composite** — multi-property bundles: `shadow-hairline` (elevation by border,
  never shadow), the `text-metric-*` tabular ramp, `text-label` (the mono label
  tier), `stale-dim`, `focus-ring`. Where Tailwind has a namespace they live in
  `@theme`; otherwise they're an `@utility`. **Every custom `text-*` composite must
  also be registered in `extendTailwindMerge`** (`packages/ui/src/lib/utils.ts`) —
  unregistered, tailwind-merge classifies it as a text *color* and silently drops
  it or the real color in `cn()` merges.

## Type

Two families, three voices — the pairing is the brand's (see
[brand](?path=/docs/system-brand--docs)):

| Voice | Utility | Face |
|---|---|---|
| Prose | `text-xs` caption → `text-sm` body → `text-base` h2 → `text-2xl` h1 | Inter |
| Numerics | `text-metric-sm / -metric / -metric-lg / -metric-xl` — always `tabular-nums` | Inter |
| Labels | `text-label` — 0.72rem, +0.08em tracking | JetBrains Mono |

Numerics never use the prose ramp (ticking numbers must not jitter); labels are not
prose (metric labels, table headers, roster headings); mono never carries digits in
data surfaces — mono digits read as code, not vitals.
