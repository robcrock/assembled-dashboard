# Token tiers — the ions

Four tiers in `packages/ui/src/styles/globals.css`, all color in **oklch**:

```
primitive ──▶ semantic ──▶ component / composite
```

Value flows left to right: each tier is minted only from the tier before it.
Consumption enters from the right — components reach the semantic and
composite/component tiers only, never a primitive (the rule below).

- **Primitive** (`--primitive-<hue>-<step>`) — context-free raw values. Defined in
  `:root` only, **never in `@theme`**: that's the enforcement trick, not a style
  preference — Tailwind generates *no utilities* for them, so a component
  structurally cannot consume a raw value by class. Never themed. Every hue ships
  the **full 50–950 scale** (Primer-style uniform ramps — a semantic role's first
  use of a step is a token pick, not a mint), and roles pick steps from documented
  usage bands: **50–200** tints, **200–300** hairlines, **400–500** anchors/fills +
  dark-theme inks (never light-theme body ink — anchors measure ~3–4:1),
  **600–800** light-theme AA text inks, **900–950** near-ink solids + dark tints.
- **Semantic** (`--background`, `--muted`, `--status-*`, `--sla-breach`…) — roles
  that reference primitives. Light/dark remaps happen at **this tier only**: `.dark`
  re-points roles; pure aliases (`--status-breached`, `--destructive`, `--ring`)
  are declared once and chase their targets through the remap.
- **Component** (`--<component>-<part>`) — the earning rule: minted only when two
  consumers need *different values for the same semantic role*. **Zero exist.** A
  knob nobody overrides is dead weight.
- **Composite** — multi-property bundles: `shadow-hairline` (elevation by border,
  never shadow), the `text-metric-*` tabular ramp, `text-label` (the mono label
  tier), `stale-dim`, `focus-ring`. Where Tailwind has a namespace they live in
  `@theme`; otherwise they're an `@utility`. **Every custom `text-*` composite must
  also be registered in `extendTailwindMerge`** (`packages/ui/src/lib/utils.ts`) —
  unregistered, tailwind-merge classifies it as a text *color* and silently drops
  it or the real color in `cn()` merges.

Today's census of the top tiers, so nobody has to grep to trust the docs:
**component tokens — zero** (the earning rule has never been met). **Composites
are all spent**: `text-metric-*` ticks in StatCard, MetricDelta, and
StaleIndicator; `text-label` heads StatCard, DataTable, and the coverage panel;
`stale-dim` dims StatCard and DataTable; `focus-ring` rides DataTable and
OrgIdentity — except `shadow-hairline`, minted and awaiting its first consumer
(the vendored Card carries its stock `ring-1` hairline: same visual voice,
stock recipe).

## The consumption rule

**Components consume tokens at the semantic tier or above — a `--primitive-*`
name never appears outside `globals.css`.** A component that needs a color asks
for the *role* (`bg-card`, `text-status-breached`), never the raw step
(`--primitive-orange-500`): the role is what `.dark` remaps and what a re-brand
re-points, so a primitive smuggled into a component is a value that silently
stops theming.

Enforcement is layered:

- **Structural** — primitives live in `:root` only, never `@theme`, so Tailwind
  generates no `bg-primitive-*` / `text-primitive-*` utilities. The class-based
  path cannot express the violation.
- **The remaining escape hatch is banned** — `var(--primitive-*)` inside an
  arbitrary value or inline style compiles fine and is therefore a review
  concern, not a compiler one. The audit is one grep, and it must stay empty:

```
grep -rn "primitive-" packages/ui/src/components packages/ui/src/lib apps/web
# → no matches
```
