# Icons — the glyph carries the verdict

One source: **lucide-react** (already the shadcn-configured set). Stroke glyphs
only — no icon font, no custom SVGs, no filled variants. And one discipline:
every icon is **functional**, never ornament. A glyph appears because it carries
meaning a scanning eye needs (a status, a sort direction, an affordance) — if
the adjacent text already says everything, the icon doesn't earn its pixels.

## The status trio

Status reads by **glyph shape first, color second** — the palette's one loud
color is reserved for breach (see the
[color law](?path=/docs/rules-color-law--docs)), so healthy and at-risk must
stay legible through shape alone. The mapping lives in exactly one place:
`STATUS_META` in `status-badge.tsx`, the single map from a status value to
glyph + label + color. It is deliberately **not exported** — no surface can
invent its own pairing — so these specimens render the real components rather
than transcribing the map:

`StatusDot` renders the standalone glyph at `size-3.5`, centered in a one-line
(`h-lh`) wrapper so it baseline-composes beside text. It announces its label to
assistive tech (`sr-only`) unless marked `decorative` — set that only when
adjacent text already names the status, so a screen reader doesn't hear it
twice.

## The rules

- **Status glyphs come only from `STATUS_META`.** No surface pairs a status
  with a glyph on its own; render `StatusDot` / `StatusBadge` and the pairing
  is structural.
- **Icons never take a color prop.** Glyphs inherit `currentColor` from their
  surface's ink; status ink comes from the canonical map, everything else
  stays in the quiet foreground/muted registers.
- **Icon-only surfaces speak.** `aria-hidden` on the SVG, an `sr-only` label
  beside it — unless the glyph is `decorative` because adjacent text already
  says it (StatusDot's contract).
- **Sizing rides the stock `size-*` steps** — `size-3` (dense table
  affordances), `size-3.5` (inline-with-text glyphs), `size-4` (buttons and
  row-level controls). No icon has ever needed a minted size token; don't be
  the first without a second consumer.
