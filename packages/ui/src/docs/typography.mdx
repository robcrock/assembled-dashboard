# Typography — two families, three voices

Inter carries prose and numerics; JetBrains Mono carries labels — the pairing is
the brand's (see [brand](?path=/docs/rules-brand--docs)). The ramp is deliberately
small: the prose steps are **stock Tailwind utilities** (no composites to learn),
and the only custom type utilities are the five composites below — four tabular
`text-metric-*` steps and the mono `text-label` tier. Hierarchy is carried by size
and weight, never by color.

Every sample on this page renders with the real utility class, so the specimens
are always the shipped truth; the printed meta is transcribed from
`packages/ui/src/styles/globals.css` — update them together.

## Faces & sourcing

| Face | Variable | Weights loaded | Carries |
| --- | --- | --- | --- |
| Inter | `var(--font-sans)` | 400 · 500 · 600 · 700 | prose + numerics |
| JetBrains Mono | `var(--font-mono)` | 400 · 500 | labels (`text-label`) |

The utilities consume only the CSS variables. The **app** defines them via
`next/font` (`apps/web/app/layout.tsx`); **Storybook** defines them in
`.storybook/preview-head.html` (Google Fonts + `:root` variables). A new face or
weight must land in **both** places, or the catalog silently falls back.

## Numeric ramp — Inter + tabular figures

The four `text-metric-*` composites. Sans + `tabular-nums`, **not** mono — mono
digits read as code, not vitals. Weight caps at 500: a number earns attention by
size, never by boldness.

### Why tabular figures

Ticking numbers must not jitter horizontally, and columns of figures must align
at the decimal. Tabular figures give every digit the same advance width:

## Label voice — JetBrains Mono

The one mono voice on the page. Labels are not prose and not numerics: small
persistent identifiers that must recede behind the data they name — which is why
the pairing with `text-muted-foreground` is part of the recipe, not optional.

## The rules

- **Numerics never use the prose ramp** — ticking numbers must not jitter; prose
  never carries tabular data.
- **Labels are always mono *and* always muted** — `text-label` +
  `text-muted-foreground` travel together.
- **Mono never carries digits in data surfaces** — mono digits read as code, not
  vitals; numbers stay Inter + `text-metric-*`.
- **Weight ceiling is 600** (`font-semibold`, headings only); the metric
  composites cap at 500. Nothing bolder, ever.
- **Every new `text-*` composite must be registered in `extendTailwindMerge`**
  (`packages/ui/src/lib/utils.ts`) or `cn()` classifies it as a text color and
  silently drops it — the long-form rationale lives in
  [tokens](?path=/docs/rules-token-tiers--docs).
- **Don't mint new ramp steps.** The monogram's `text-xl` is the only shipped
  off-ramp, and it needed a reason.

## Choosing a voice

- A number the eye compares or that ticks → `text-metric-*`.
- A small persistent identifier (metric label, column header, group heading) →
  `text-label` + `text-muted-foreground`.
- Everything else is prose → the stock ramp, `text-xs` → `text-2xl`.

For the component-level version of this decision, see
[choosing-a-data-primitive](?path=/docs/skills-choosing-a-data-primitive--docs).
