# Column types — one declaration, both faces

A **column type** is a resolver from one value shape to everything a cell
needs: the **view** (a display primitive), the **editor** (an editor
primitive over the *same* value — where the value is legitimately editable),
a default **sort projection**, and the draft adaptation between them. A
column declares `type` + `get` instead of hand-wiring a `cell`:

```tsx
const col = createColumns<Agent>()

const columns = [
  col.text({ key: "agent", header: "Agent", get: (a) => a.name }),
  col.enum({ key: "state", header: "State", get: (a) => a.state, options: STATE_OPTIONS }),
  col.duration({ key: "out-for", header: "Out for", get: (a) => a.out_of_adherence_sec }),
]
```

The point of routing both faces through one declaration: the view and the
editor **cannot drift apart** — they speak the same value by construction,
and the interaction layer resolves a cell's inline editor from the same
source. Editing is one gesture — click the cell you mean; there is
deliberately no row menu and no batched row form.

Two kinds of type, one grammar — every type is a factory call:

- **value primitives** — `text`, `number`, `enum`, `multiselect`, `duration`;
- **preferred components** — `status` (StatusBadge · EnumSelect), `delta`
  (MetricDelta · NumberFieldEditor), and the **view-only** `sparkbars`,
  `deviation`, `meter`, which visualize observations and deliberately expose
  no edit face.

## The rules the system holds

- **Precedence:** a column's own `cell` beats `type.view`, and an `edit`
  binding's own `type` beats the column's. The escape hatches exist for
  anatomies richer than one value — the queue table's compound headroom cell
  stays a `cell`, and pairs `editCell` (how the cell rests in edit mode) with
  `renderField` (the read-only context wrapped around the live editor) so
  `48s / [3m]` survives the click.
- **Sort follows meaning, not the alphabet.** A type's `sortValue` is a
  convenience the column's own `sortValue` overrides. `enum` sorts by
  **declaration order** (declare the semantic order); `status` sorts by
  severity — breached first under the triage convention.
- **Drafts are real states.** `number` and `duration` edit through
  `number | null` — an emptied field is a draft, not zero — with
  `toDraft`/`fromDraft` adapting at the boundary. `fromDraft` returning
  `null` means "not committable as is"; enforcement is the interaction
  layer's job.
- **View value ≠ edit value.** A derived display (the status badge computed
  from a target) is a read-only projection; its column binds
  `edit: { field, get, type }` to the *config field behind it* instead.
  That's the write inverse a computed `get` cannot provide.
- **The registry is open.** ui ships the types whose views + unions live
  here; a slice mints its own named types from the same factories —
  `defineColumnTypes({ agentState: columnTypes.enum(AGENT_STATE_OPTIONS) })`
  — and passes them through `col.custom(...)`. Options are often live domain
  data (an agent's assignable queues); thread them in like `queueNamesById`.

## Deliberately omitted

- **`date`/`boolean` types** — no consumer yet; a type is minted on first
  use, not speculatively.
- **A string-keyed global registry** — columns reference `ColumnType`
  *objects*, so there is no hidden lookup table to drift and DataTable needs
  no registry prop; "names" are just exported bindings.
- **Editor faces on `sparkbars`/`deviation`/`meter`** — they visualize
  observations; an editable observation is a lie about the floor.
